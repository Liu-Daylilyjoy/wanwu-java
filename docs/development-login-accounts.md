# Development Login Accounts

Date: 2026-06-30

The Docker development environment has two built-in IAM accounts. They are code-backed development accounts, not the final reproduced IAM user/role persistence model.

| Username | Token | Permission scope | Intended check |
| --- | --- | --- | --- |
| `admin` | `dev-token` | `permission`, `permission.user`, `permission.org`, `permission.role`, `setting`, `model`, `model.model_management`, `app`, `app.rag`, `app.workflow`, `app.agent`, `api_key`, `api_key.api_key_management`, `resource`, `resource.knowledge`, `resource.tool`, `resource.mcp`, `resource.prompt`, `resource.skill`, `resource.safety`, `operation`, `operation.oauth`, `operation.statistic_client`, `exploration`, `exploration.app`, `exploration.mcp`, `exploration.template`, `exploration.skill`, `app_observability`, `app_observability.statistic` | Verify permission-management read views, platform settings, Operation Management, Exploration Square, App Observation, Model Management, Knowledge, Tool, MCP, Prompt, Skill, Safety Guard, RAG, Workflow, Agent, and API Key menus without exposing unreproduced modules. |
| `app` | `dev-token-app` | `app`, `app.rag`, `app.workflow`, and `app.agent` only | Verify the application-development-only view. |

Password and captcha are development placeholders. Use any non-empty password, such as `x`, and captcha code `1234`.

Frontend permissions are intentionally narrower than `web/src/router/constants.js` while the Java reproduction is still incomplete. Unreproduced modules stay hidden until their backend slices are implemented, tested, and verified through Docker Compose. Permission management currently covers the read-only user, role, organization, role-select, and role-template views; write operations are still a later IAM persistence slice. Platform settings currently cover tab, login, and home custom configuration writes plus `/base/custom` readback using the Docker development IAM repository; Operate-service persistence and uploaded asset storage are still later. Operation Management currently covers OAuth app create/list/update/delete/status with a Docker development IAM repository and `/statistic/client` empty overview/trend data for the frontend statistics tab; OAuth authorize/token runtime and real Operate/Redis aggregation remain later. Exploration Square currently covers app square list/favorite/history contracts, MCP square, prompt template square, and Skill square contracts using existing App/MCP development repositories; chatflow marketplace routes and real history ranking remain later. App Observation currently covers app/model/API Key statistic selects, overview cards, trend chart contracts, tables, details, and export button compatibility with zero-valued development data; real Operate/App aggregation remains later. Model Management currently covers the model list, detail, import/update/delete/status, provider list, recommendation list, and model select endpoints with a Docker development in-memory repository. Knowledge currently covers knowledge base create/list/update/delete, tag binding, splitter lists, document empty-state views, metadata shells, and permission compatibility using a Docker development in-memory repository. Tool/MCP/Prompt/Skill currently cover resource-center CRUD/list/detail/select/action/config contracts, local OpenAPI schema parsing, MCP Server tool binding, local MCP and Skill square seeds, deterministic Prompt SSE optimize/reason/evaluate responses, and deterministic Skill conversation SSE/save responses using the Docker development `wanwu-service-mcp` repository. Safety Guard currently covers sensitive table/word management and the Agent/RAG safety table selector using the Docker development `wanwu-service-app` repository; runtime stream interception is still later. RAG currently covers the frontend management lifecycle plus draft/published chat shell and multipart upload response compatibility; real retrieval/generation is still later. Workflow currently covers appspace lifecycle, import/export, publish/version, local run shell, `/workflow/api` compatibility, and avatar upload compatibility; the real workflow editor/runtime remains later. The ontology agent menu has been removed from this reproduction scope.

Current limitation: BFF business write operations still map request context to the shared development organization and user until the full IAM user/org/role persistence slice is reproduced.
