# Development Login Accounts

Date: 2026-06-30

The Docker development environment has two built-in IAM accounts. They are seed accounts plus a MySQL-backed development compatibility repository, not the final reproduced IAM user/role relational model.

| Username | Token | Permission scope | Intended check |
| --- | --- | --- | --- |
| `admin` | `dev-token` | `permission`, `permission.user`, `permission.org`, `permission.role`, `setting`, `model`, `model.model_management`, `app`, `app.rag`, `app.workflow`, `app.agent`, `api_key`, `api_key.api_key_management`, `resource`, `resource.knowledge`, `resource.tool`, `resource.mcp`, `resource.prompt`, `resource.skill`, `resource.safety`, `operation`, `operation.oauth`, `operation.statistic_client`, `exploration`, `exploration.app`, `exploration.mcp`, `exploration.template`, `exploration.skill`, `app_observability`, `app_observability.statistic` | Verify permission-management read views, platform settings, Operation Management, Exploration Square, App Observation, Model Management, Knowledge, Tool, MCP, Prompt, Skill, Safety Guard, RAG, Workflow/Chatflow, Agent, and API Key menus without exposing unreproduced modules. |
| `app` | `dev-token-app` | `app`, `app.rag`, `app.workflow`, and `app.agent` only | Verify the application-development-only view, including Workflow and Chatflow tabs behind the original `app.workflow` frontend permission. |

Password and captcha are development placeholders. Use any non-empty password, such as `x`, and captcha code `1234`.

Frontend permissions are intentionally narrower than `web/src/router/constants.js` while the Java reproduction is still incomplete. Unreproduced modules stay hidden until their backend slices are implemented, tested, and verified through Docker Compose. Common user/profile routes currently cover development user info, password/avatar/language success contracts, and Doc Center seed content; real IAM profile/password policy remains later. Permission management currently covers user, role, organization, role-select, role-template, write operations, status changes, and multipart batch-import compatibility; mutable IAM/OAuth/custom-setting records survive restarts through `iam_service.iam_records`, while normalized Go-equivalent IAM tables, invitations, audit logs, and real Excel parsing remain later. Platform settings currently cover tab, login, and home custom configuration writes plus `/base/custom` readback using the Docker development IAM repository; Operate-service uploaded asset storage is still later. Operation Management currently covers OAuth app create/list/update/delete/status with the Docker development IAM repository and `/statistic/client` empty overview/trend data for the frontend statistics tab; OAuth authorize/token runtime and real Operate/Redis aggregation remain later. Exploration Square currently covers app square list/favorite/history contracts, MCP square, prompt template square, Skill square, assistant template, and workflow template contracts using existing App/MCP development repositories plus local template seeds; real template ranking and real history ranking remain later. App Observation currently covers app/model/API Key statistic selects, overview cards, trend chart contracts, tables, details, and export button compatibility with zero-valued development data; real Operate/App aggregation remains later. Model Management currently covers the model list, detail, import/update/delete/status, provider list, recommendation list, model select endpoints, and model-experience history with a Docker MySQL compatibility repository. Knowledge currently covers knowledge base create/list/update/delete, keyword mappings, tag binding, splitter lists, document empty-state views, local document/segment state, metadata shells, permission compatibility, and QA pairs using a Docker MySQL snapshot repository. File upload currently covers chunk upload/check/list/merge/clean/delete, direct upload, proxy upload, inferpub upload, and OpenURL anonymous upload using BFF-local development storage; MinIO/object lifecycle parity remains later. Public OpenAPI currently covers `/service/api/openapi/v1` agent, RAG, workflow, chatflow, model, knowledge, MCP, and OAuth route shapes with development API-key context; real inference, OAuth signing, workflow execution, MCP runtime, knowledge indexing, and API usage metrics remain later. Tool/MCP/Prompt/Skill currently cover resource-center CRUD/list/detail/select/action/config contracts, local OpenAPI schema parsing, MCP Server tool binding, local MCP and Skill square seeds, deterministic Prompt SSE optimize/reason/evaluate responses, and deterministic Skill conversation SSE/save responses using a Docker MySQL snapshot repository in `wanwu-service-mcp`. Safety Guard currently covers sensitive table/word management and the Agent/RAG safety table selector using a Docker MySQL snapshot repository in `wanwu-service-app`; runtime stream interception is still later. RAG currently covers the frontend management lifecycle plus draft/published chat shell and multipart upload response compatibility; real retrieval/generation is still later. Workflow and Chatflow currently cover appspace lifecycle, template copy, import/export, publish/version, local run shell, `/workflow/api` compatibility, chatflow application compatibility, and avatar upload compatibility; the real workflow/chatflow editor/runtime remains later. The ontology agent menu has been removed from this reproduction scope.

Current limitation: BFF business write operations still map request context to the shared development organization and user until full token-to-user/org resolution is reproduced.
